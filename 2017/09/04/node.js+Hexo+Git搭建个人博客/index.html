node.js+Hexo+Git搭建博客

node.js+Hexo+Git

Posted by Highya Guo on 2017-09-04

其实很早之前我就已经搭建了git博客，但是谁叫我是懒癌晚期呢，无赖一直没写东西，哈哈哈～ 以前都是用的云笔记一类的，但是觉得归档特别麻烦，最近突然想开始整理一些知识点，当即决定还是要有自己的一个博客网站，于是就这样开始了～～

本来买好了域名，准备用node.js+Express+Mongodb搭建一个个人网站，但是关于数据库，服务端这一块的东西自己并不是很擅长，再加上最近时间有限，暂时还是用之前用过的方案node.js+Hexo+Git。下面是我简单记录一下搭建流程：

node.js+Hexo+Git搭建个人博客

准备工作

1、安装

  • Node.js
  • Git

2、注册账号（github）

GitHub官网注册一个账号，然后创建一个新的仓库（repository），仓库的名字一定要类似于：userName.github.io，userName一定要和你创建的Git账户名字相一致，所以取名字要慎重哦，自己喜欢就好啦～

Hexo的安装与配置

1、npm安装Hexo

$ npm install -g hexo-cli

2、初始化

$ hexo init <folder>     -> 例：$ hexo init userName.github.io
$ cd <folder>
$ npm install

3、配置

找到根目录下_config.yml文件进行配置，一般我们需要修改的有：title、author、language(汉语：zh-Hans)、theme(要使用的主题名字)、deploy(部署)下的类型（type:本例写git)和仓库（repo:https://github.com/userName.github.io.git）

4、启动服务

$ hexo server

在终端输入上面的命令，正常的情况下会出现：INFO Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.，这表明配置已经正确，hexo开始运行了，接下来我们只需在浏览器中输入http://localhost:4000/，不出意外就可以看到Hello world的页面了。

5、更换主题

更换主题很简单， 可以直接去Hexo官网去选一个自己喜欢的主题下载， 然后把它放在根目录的themes文件夹下，最后一步就是更改配置文件_config.yml中theme(主题名字)。

调试并上传

1、本地调试

$ hexo clean && hexo s   -> 注:clean是清除缓存

其实在上一步我们已经看到本地调试的效果啦～ 对！就是Hello world， 哈哈哈哈～～

2、上传到github仓库

$ npm install hexo-deployer-git --save
$ hexo clean && hexo g && hexo d

本地调试好之后，接下来就是见证奇迹的时候了，我们想访问自己的userName.github.io的时候也能看到这样的效果。那么只需要把本地的文件上传至gitHub服务器即可。这里我们先通过：$ npm install hexo-deployer-git –save安装自动部署发布工具，然后输入$ hexo clean && hexo g && hexo d，等待系统上传完成即可。注：hexo g表示generate，生成静态文件；hexo d表示deploy，部署文件。
